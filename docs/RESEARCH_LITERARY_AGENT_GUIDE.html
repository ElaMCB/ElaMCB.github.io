Research & Literary Agent

Your automated research and publishing agent. It runs monthly to curate discoveries and update the State of AI Testing doc so you don’t have to initiate research or publish by hand.

What it does

  • Research: Uses your latest llm-discovery/ markdown file (e.g. discoveries-YYYY-MM-DD.md) as the source of LLM/Gen AI discoveries.
  • Publish: Inserts a new month section into docs/STATE_OF_AI_TESTING.html with curated items (title, description, link), so the page grows automatically.
  • Schedule: Runs on the 1st of every month at 09:00 UTC. On that run it publishes the previous month’s section (e.g. run on 1 April → section “March 2026”).

How to run it

  • Automatic: Nothing to do; the workflow runs on the 1st of each month.
  • Manual: Go to Actions → Research & Literary Agent → Run workflow. Optionally check “Only run update, do not commit/push” for a dry run.

What it needs from you

  • Discovery files: Add (or have another process add) llm-discovery/discoveries-YYYY-MM-DD.md with the usual structure: ### N. Title, - **Description**:, - **URL**:. The agent picks the latest file by modification time.
  • If no discovery file exists for the period, the agent still adds a month section with a short “no discoveries” message so the doc stays up to date.

Files and workflow

  • Updated doc: State of AI Testing – broad-titled living doc; no “Q1 2026” in the name.
  • Workflow: .github/workflows/research-literary-agent.yml
  • Script: agents/capabilities/research-literary/monthly_update.py
